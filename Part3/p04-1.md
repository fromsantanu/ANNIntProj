## Chapter 10.1: Accuracy

Accuracy is the most basic and commonly used metric to evaluate a model.

---

### 🧠 What is Accuracy?

Accuracy means:

> “Out of all predictions, how many are correct?”

---

### 📊 Formula

\text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Total Predictions}}

---

### 🧪 Simple Example

Suppose:

* Total customers = 100
* Correct predictions = 85

👉 Accuracy = **85%**

---

### 🧩 Understanding with Churn Example

| Actual | Predicted | Result  |
| ------ | --------- | ------- |
| 1      | 1         | Correct |
| 0      | 0         | Correct |
| 1      | 0         | Wrong   |
| 0      | 1         | Wrong   |

👉 Accuracy counts **only correct predictions**

---

### ⚙️ Python Code Example

```python id="xk6u2f"
from sklearn.metrics import accuracy_score

# Convert probabilities to 0 or 1
y_pred = (model(X_test) > 0.5).float()

accuracy = accuracy_score(y_test, y_pred)
print("Accuracy:", accuracy)
```

---

### ⚠️ Important Limitation

Accuracy can be misleading in some cases.

---

### 🧪 Example Problem

Suppose:

* 90 customers stay
* 10 customers leave

Model predicts:

* Everyone will stay

👉 Accuracy = 90%

But:

* It failed to detect any churn

👉 So the model is **not useful**

---

### 📌 When Accuracy Works Well

✔ When data is balanced
✔ When both classes are equally important

---

### 📈 Real-Life Analogy

Imagine an exam:

* 100 questions
* You answer 90 correctly

👉 Accuracy = 90%

But if:

* All difficult questions are wrong

👉 Your understanding may still be weak

---

### 🚀 Key Takeaways

✔ Accuracy is simple and easy to understand
✔ It measures overall correctness
✔ Not reliable when data is imbalanced
