## Chapter 7.1: Model Architecture

In this section, we focus only on **how the ANN is structured**.

👉 Think of architecture as the **blueprint of the model**.

---

### 🧠 What is Model Architecture?

Model architecture means:

> “How many layers are there, and how many neurons are in each layer”

---

### 🧩 Our ANN Structure

For this project, we will use the following structure:

* **Input Layer** → 10 features
* **Hidden Layer 1** → 64 neurons
* **Hidden Layer 2** → 32 neurons
* **Hidden Layer 3** → 16 neurons
* **Output Layer** → 1 neuron

---

### 📊 Visual Understanding

```
Input (10)
   ↓
Hidden Layer (64)
   ↓
Hidden Layer (32)
   ↓
Hidden Layer (16)
   ↓
Output (1)
```

---

### 🎯 Why This Structure?

Let’s understand simply:

#### 🔹 Input Layer

* Receives customer data
* Example: age, bill, complaints

👉 One neuron per feature

---

#### 🔹 Hidden Layers

* These layers **learn patterns**
* Each layer extracts deeper information

👉 Example:

* Layer 1 → basic patterns
* Layer 2 → more complex patterns
* Layer 3 → refined understanding

---

#### 🔹 Output Layer

* Gives final prediction

👉 One neuron → because output is:

* 0 (stay)
* 1 (leave)

---

### 🧠 Why Decreasing Neurons?

We go from:

64 → 32 → 16

👉 This is like:

* First layer → broad understanding
* Later layers → more focused decisions

---

### ⚖️ How Many Layers Should We Use?

There is no fixed rule, but:

* Too few layers → model too simple
* Too many layers → overfitting (memorizing data)

👉 Our design is a **balanced approach**

---

### ⚙️ Architecture in PyTorch

```python id="fj4mh2"
nn.Sequential(
    nn.Linear(10, 64),
    nn.ReLU(),

    nn.Linear(64, 32),
    nn.ReLU(),

    nn.Linear(32, 16),
    nn.ReLU(),

    nn.Linear(16, 1),
    nn.Sigmoid()
)
```

---

### 📈 Real-Life Analogy

Think of solving a problem:

1. First you gather all information
2. Then you analyze step by step
3. Finally, you make a decision

👉 Each step = one layer in ANN

---

### 🚀 Key Takeaways

✔ Architecture defines how the model thinks
✔ More layers → deeper learning
✔ Balanced design gives better results
