## Chapter 9.2: Validation Step

After training the model, we need to check:

👉 **Is the model really learning, or just memorizing?**

This is where the **Validation Step** comes in.

---

### 🧠 What is Validation?

Validation means:

> “Testing the model on unseen data during training”

---

### 🎯 Why Do We Need Validation?

If we only check training performance:

👉 The model may look good but fail in real-world data.

Validation helps us:

* Detect overfitting
* Monitor performance
* Make better decisions

---

### 🧩 Data Split Reminder

We usually divide data into:

* Training set → used to learn
* Validation set → used to check during training
* Test set → final evaluation

👉 Validation acts like a **practice test**

---

### 🔍 What Happens in Validation?

During validation:

* We pass validation data through the model
* We calculate loss or accuracy
* **No learning happens here**

👉 Model weights are NOT updated

---

## ⚙️ Validation in PyTorch

---

### 🧪 Example Code

```python id="6h6h4g"
model.eval()  # Set model to evaluation mode

with torch.no_grad():  # Disable gradient calculation
    
    val_predictions = model(X_test)
    val_loss = criterion(val_predictions, y_test)

print(f"Validation Loss: {val_loss.item():.4f}")
```

---

### 🧠 Important Points

---

#### 🔹 `model.eval()`

* Turns off dropout
* Uses full network

👉 Ensures proper evaluation

---

#### 🔹 `torch.no_grad()`

* Stops gradient calculation
* Saves memory and time

👉 We don’t need learning here

---

## 📊 Comparing Training vs Validation

| Type       | Purpose           | Learning |
| ---------- | ----------------- | -------- |
| Training   | Learn patterns    | Yes      |
| Validation | Check performance | No       |

---

### ⚠️ Detecting Overfitting

If:

* Training loss ↓ (good)
* Validation loss ↑ (bad)

👉 Model is overfitting

---

### 📈 Real-Life Analogy

Imagine preparing for an exam:

* Study → training
* Practice test → validation
* Final exam → testing

👉 Practice test shows your real readiness

---

### 🚀 Key Takeaways

✔ Validation checks real performance
✔ No weight updates happen
✔ Helps detect overfitting early
✔ Essential for building a reliable model
