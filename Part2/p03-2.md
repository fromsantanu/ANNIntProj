## Chapter 5.2: Encoding Categorical Variables

In our dataset, some columns contain **text values** like:

* “Monthly”, “Yearly”
* “UPI”, “Card”, “Cash”

👉 These are called **categorical variables**

---

### 🧠 Why Encoding is Needed

ANN models (built using PyTorch) **cannot understand text directly**.

👉 They only understand numbers.

So we must convert text into numbers.

> This process is called **Encoding**

---

### 🧾 Example Before Encoding

| Contract Type | Payment Method |
| ------------- | -------------- |
| Monthly       | UPI            |
| Yearly        | Card           |
| Monthly       | Cash           |

---

### 🎯 Goal

Convert this into a **numeric format** that the model can use.

---

## 🛠️ Encoding Methods

---

### 🔹 1. Label Encoding

Each category is assigned a number.

#### Example:

| Contract Type | Encoded |
| ------------- | ------- |
| Monthly       | 0       |
| Yearly        | 1       |

👉 Simple replacement.

---

### ⚠️ Problem with Label Encoding

The model may think:

> Yearly (1) > Monthly (0)

👉 But in reality, there is **no such order**

---

### 🔹 2. One-Hot Encoding (Most Important)

We create **separate columns** for each category.

#### Example:

| Contract Type | Monthly | Yearly |
| ------------- | ------- | ------ |
| Monthly       | 1       | 0      |
| Yearly        | 0       | 1      |

👉 No false ordering → safer for ANN

---

### 🔹 Example for Payment Method

| Payment | UPI | Card | Cash |
| ------- | --- | ---- | ---- |
| UPI     | 1   | 0    | 0    |
| Card    | 0   | 1    | 0    |

---

### 📌 When to Use What?

| Method           | When to Use                                  |
| ---------------- | -------------------------------------------- |
| Label Encoding   | When order matters (e.g., Low, Medium, High) |
| One-Hot Encoding | When no order exists (most cases)            |

---

### ⚙️ Code Example

```python id="cwrrqm"
import pandas as pd

# One-hot encoding
df = pd.get_dummies(df, columns=['Contract', 'Payment'])
```

---

### 🧠 What Happens After Encoding?

All columns become numeric:

✔ Ready for ANN
✔ No text remains
✔ Model can process easily
