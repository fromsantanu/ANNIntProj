## Chapter 4: Exploratory Data Analysis (EDA)

Before training our model, we need to **look closely at the data** and understand its patterns.

👉 This step is called **Exploratory Data Analysis (EDA)**.

---

### 🧠 What is EDA?

EDA means:

> “Exploring the data to understand what is going on inside it”

Instead of directly building a model, we first:

* Observe patterns
* Find problems
* Get useful insights

---

### 🎯 Why EDA is Important

If we skip EDA:

* We may use bad data
* The model may learn wrong patterns

👉 EDA helps us **avoid mistakes early**

---

### 🔍 What Do We Look For?

#### 🔹 1. Distribution of Data

We check how values are spread.

Example:

* Are most customers young or old?
* Are bills mostly high or low?

👉 This helps us understand “normal behavior”

---

#### 🔹 2. Churn Distribution

We check:

* How many customers stayed?
* How many customers left?

👉 Example:

* 90% stay, 10% leave → dataset is imbalanced

This is important for model training.

---

#### 🔹 3. Relationship Between Features

We try to see:

* Do high bills lead to churn?
* Do more complaints increase churn?

👉 Like finding hidden connections

---

#### 🔹 4. Outliers (Strange Values)

Example:

* Monthly bill = ₹1,00,000
* Age = 150

👉 These are unrealistic and can confuse the model.

---

#### 🔹 5. Missing Values

We check:

* Are some values empty or null?

👉 Example:

* Missing payment method
* Missing complaint count

---

### 📊 Simple Visualizations

We use graphs to understand data easily:

* Bar charts → compare categories
* Histograms → see distribution
* Box plots → detect outliers

👉 Think of graphs as “pictures of data”

---

### ⚙️ Tools We Will Use

* **Pandas** → to inspect data
* **Matplotlib / Seaborn** → to create graphs
* Later we move to PyTorch for modeling

---

### 🧪 Example Questions We Try to Answer

* Do customers with monthly contracts churn more?
* Does higher complaint count increase churn?
* Are high-paying customers more loyal?

👉 These insights guide model design.

---

### 📈 Real-Life Analogy

Imagine a doctor:

Before giving treatment, the doctor:

* Checks reports
* Observes symptoms
* Looks for patterns

👉 EDA is like diagnosing before treatment.

---

### 🚀 Outcome of EDA

After EDA, we will:

✔ Understand data structure
✔ Identify issues
✔ Know which features are important
✔ Be ready for preprocessing
