## Chapter 3: Understanding the Dataset

Before building any model, we must clearly understand the **data we are working with**.

👉 A good model always starts with good understanding of data.

---

### 📊 What is a Dataset?

A dataset is simply a **collection of rows and columns**.

* Each **row** = one customer
* Each **column** = one type of information

👉 Think of it like an Excel sheet.

---

### 🧾 Example of Our Dataset

| Age | Monthly Bill | Complaints | Contract Type | Payment Method | Churn |
| --- | ------------ | ---------- | ------------- | -------------- | ----- |
| 45  | 2000         | 3          | Monthly       | Credit Card    | 1     |
| 30  | 800          | 0          | Yearly        | UPI            | 0     |

---

### 🧠 Understanding Each Column

Let’s break it into simple terms:

#### 🔹 Numerical Features (Numbers)

* Age
* Monthly Bill
* Number of Complaints

👉 These are already numbers, easy for the model to understand.

---

#### 🔹 Categorical Features (Text)

* Contract Type (Monthly / Yearly)
* Payment Method (UPI / Card / Cash)

👉 These are words → we will convert them into numbers later.

---

#### 🔹 Target Column (What We Predict)

* **Churn**

Values:

* `0 → Stay`
* `1 → Leave`

👉 This is the **answer column** the model will learn from.

---

### 📌 Types of Data in Simple Words

| Type        | Example      | Meaning              |
| ----------- | ------------ | -------------------- |
| Numerical   | 45, 2000     | Measured values      |
| Categorical | Monthly, UPI | Labels or categories |
| Target      | 0 / 1        | Final output         |

---

### 🔍 Why Understanding Data is Important

Imagine teaching a student without explaining the topic clearly.

👉 Same happens with ANN if data is not understood.

If we misunderstand:

* Model may learn wrong patterns
* Predictions become poor

---

### 🧪 Basic Checks We Should Do

Before using the data, we check:

* Are there missing values?
* Are values in correct format?
* Are there extreme or wrong values?

👉 Example:
If age = 500 → clearly wrong
