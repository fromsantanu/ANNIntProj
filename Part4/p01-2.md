## Chapter 11.2: Trying Different Architectures

In the previous section, we tuned parameters like learning rate and epochs.
Now we focus on another powerful idea:

👉 **Changing the model architecture**

---

### 🧠 What is Architecture Tuning?

Architecture tuning means:

> “Changing the structure of the neural network to improve performance”

---

### 🎯 What Can We Change?

We can experiment with:

* Number of layers
* Number of neurons in each layer
* Arrangement of layers
* Adding/removing dropout

---

## 🔍 Why Try Different Architectures?

Different problems need different designs.

👉 A simple model may:

* Miss important patterns (underfitting)

👉 A very complex model may:

* Memorize data (overfitting)

---

## 🛠️ Common Architecture Variations

---

### 🔹 1. Increase Number of Layers

**Before:**

* 2 hidden layers

**After:**

* 3 or 4 hidden layers

👉 Helps capture deeper patterns

---

### 🔹 2. Increase Number of Neurons

**Before:**

* 64 → 32 → 16

**After:**

* 128 → 64 → 32

👉 Gives more learning capacity

---

### 🔹 3. Reduce Model Size

Sometimes smaller models work better.

👉 Example:

* 32 → 16 → 8

✔ Faster
✔ Less overfitting

---

### 🔹 4. Add Dropout

👉 Helps control overfitting

Example:

* Add `Dropout(0.3)` between layers

---

### 🔹 5. Change Layer Pattern

Instead of always decreasing:

👉 Try different patterns:

* 64 → 64 → 32
* 128 → 64 → 16

---

## ⚙️ Example Architectures in PyTorch

---

### 🧪 Model 1 (Baseline)

```python id="fb5jhk"
nn.Sequential(
    nn.Linear(10, 64),
    nn.ReLU(),
    nn.Linear(64, 32),
    nn.ReLU(),
    nn.Linear(32, 1),
    nn.Sigmoid()
)
```

---

### 🧪 Model 2 (Deeper Network)

```python id="h2h8xn"
nn.Sequential(
    nn.Linear(10, 128),
    nn.ReLU(),
    nn.Dropout(0.3),

    nn.Linear(128, 64),
    nn.ReLU(),

    nn.Linear(64, 32),
    nn.ReLU(),

    nn.Linear(32, 1),
    nn.Sigmoid()
)
```

---

### 🧪 Model 3 (Smaller Network)

```python id="v7n1pk"
nn.Sequential(
    nn.Linear(10, 32),
    nn.ReLU(),

    nn.Linear(32, 16),
    nn.ReLU(),

    nn.Linear(16, 1),
    nn.Sigmoid()
)
```

---

### 📊 How to Compare?

After trying each model:

* Train it
* Evaluate using accuracy, precision, recall, F1

👉 Choose the best performing one

---

### 📌 Important Tip

👉 Do not change everything at once

Change:

* Only architecture
* Keep other settings same

👉 This helps you understand what works

---

### 📈 Real-Life Analogy

Think of building a house:

* Small house → simple but limited
* Big house → powerful but expensive
* Best house → balanced design

👉 Same with neural networks

---

### 🚀 Key Takeaways

✔ Architecture strongly affects performance
✔ Try different designs and compare
✔ Balance between simple and complex
✔ Combine with hyperparameter tuning
