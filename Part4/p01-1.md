## Chapter 11.1: Hyperparameter Tuning

Now that we know how to improve a model, let’s focus on one of the most important techniques:

👉 **Hyperparameter Tuning**

---

### 🧠 What are Hyperparameters?

Hyperparameters are:

> “Settings of the model that we choose before training”

---

### 🎯 Examples of Hyperparameters

* Learning rate
* Number of epochs
* Batch size
* Number of layers
* Number of neurons
* Dropout rate

👉 These are **not learned by the model**
👉 We must choose them carefully

---

### 🔍 Why Tuning is Important

Wrong values can cause:

* Slow learning
* Poor accuracy
* Overfitting or underfitting

👉 Good tuning = better performance

---

## 🛠️ Important Hyperparameters to Tune

---

### 🔹 1. Learning Rate

Controls how fast the model learns.

#### 🎯 Example:

* `0.01` → fast but unstable
* `0.001` → balanced (commonly used)
* `0.0001` → slow but stable

---

### 🔹 2. Number of Epochs

How many times the model sees the data.

#### 🎯 Example:

* Too few → underfitting
* Too many → overfitting

---

### 🔹 3. Batch Size

Number of samples used in one update.

#### 🎯 Example:

* Small batch → slower but more accurate
* Large batch → faster but less detailed

---

### 🔹 4. Number of Neurons / Layers

Controls model complexity.

#### 🎯 Example:

* More neurons → more power
* Too many → overfitting

---

### 🔹 5. Dropout Rate

Controls how many neurons are turned off.

#### 🎯 Example:

* 0.2 → light dropout
* 0.5 → strong dropout

---

## ⚙️ Simple Tuning Strategy

👉 Try different values and compare results

Example:

| Learning Rate | Accuracy |
| ------------- | -------- |
| 0.01          | 78%      |
| 0.001         | 85%      |
| 0.0001        | 82%      |

👉 Best choice → `0.001`

---

## 🧪 Code Example (Manual Tuning)

```python id="y9c2bp"
for lr in [0.01, 0.001, 0.0001]:
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    
    # Train model (simplified)
    for epoch in range(10):
        outputs = model(X_train)
        loss = criterion(outputs, y_train)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f"Learning Rate: {lr}, Loss: {loss.item():.4f}")
```

---

## 📌 Important Tip

👉 Change **one parameter at a time**

Otherwise:

* You won’t know which change improved the model

---

## 📈 Real-Life Analogy

Think of cooking:

* Salt amount → learning rate
* Cooking time → epochs
* Ingredients → features

👉 You adjust step by step to get the best taste

---

## 🚀 Key Takeaways

✔ Hyperparameters control how the model learns
✔ They must be chosen carefully
✔ Try multiple values and compare
✔ Small changes can make big differences
